# TDD Critic Rubric

Score every item. Any BLOCKER → verdict is "revise".

1.  Problem clarity       Can a cold reader understand the problem without prior context?
2.  Evidence              Is there concrete proof this is a real problem?
3.  Impact                Is severity/scope quantified or bounded?
4.  Alternatives          Are 3+ options genuinely represented (not strawmanned)?
5.  Tradeoff honesty      Is "do nothing" considered? Are the chosen option's downsides stated?
6.  Recommendation        Does the recommendation follow from the analysis?
7.  Falsifiability        Could a reader identify when the recommendation would be wrong?
8.  Implementation        Are API/schema/dependency changes identified at a high level?
9.  Scope                 Is out-of-scope explicitly stated?
10. Consistency           Does the solution actually solve the stated problem?
11. UA completeness       BLOCKER if any timed-out question is absent from
                          ## Unresolved Ambiguities. Each entry must have:
                          all options listed, chosen default marked,
                          rationale verbatim, "To override" instructions.
